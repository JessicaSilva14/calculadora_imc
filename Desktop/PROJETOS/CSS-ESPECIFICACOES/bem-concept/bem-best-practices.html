✨ BEM - Boas Práticas

Diretrizes para Implementação Eficiente

Seguir boas práticas garante que seu código BEM seja sustentável, escalável e fácil de manter ao longo do tempo.

Princípios Gerais

  • ✓ Mantenha consistência: Use sempre a mesma convenção de nomenclatura em todo o projeto
  • ✓ Seja descritivo: Nomes de classes devem ser auto-explicativos sobre sua função
  • ✓ Evite profundidade excessiva: Não crie hierarquias muito profundas de elementos
  • ✓ Documente seu código: Comente blocos complexos e decisões importantes
  • ✓ Use ferramentas: Considere pré-processadores como Sass para nesting controlado

Nomenclatura

Blocos

  • ✓ Use nomes específicos: user-profile ao invés de profile
  • ✓ Separe palavras com hífen: search-form, product-card
  • ✗ Evite abreviações obscuras: Prefira button ao invés de btn
  • ✗ Não use camelCase: Use user-profile, não userProfile

Elementos

  • ✓ Seja específico sobre a função: search-form__input, product-card__title
  • ✓ Use substantivos descritivos: __button, __title, __content
  • ✗ Evite elementos genéricos: Não use __div ou __span
  • 💡 Elementos podem ter sub-elementos: menu__item__link (use com moderação)

Modificadores

  • ✓ Prefira booleanos: --disabled, --active, --featured
  • ✓ Seja semântico: --large, --small, --primary
  • ✗ Evite valores específicos: Prefira --large ao invés de --width-200
  • 💡 Use modificadores de elemento: product-card__title--large

Estrutura e Organização

Arquivos CSS

  • ✓ Separe por categoria: base.css, layout.css, module.css, theme.css
  • ✓ Use imports: @import url('layout.css'); para organização modular
  • ✓ Minifique para produção: Use ferramentas para otimizar CSS final

Hierarquia de Elementos

  • ✓ Mantenha shallow hierarchy: Evite mais de 2-3 níveis de profundidade
  • ✓ Quebre blocos grandes: Se um bloco tem muitos elementos, considere dividi-lo
  • ✗ Evite hierarquias profundas: page__section__article__content__text__link

Exemplos Práticos

✅ Estrutura Recomendada

/* Estrutura organizada */ src/ ├── css/ │ ├── base.css /* Reset, tipografia base */ │ ├── layout.css /* Grid, containers */ │ ├── modules/ /* Componentes */ │ │ ├── button.css │ │ ├── card.css │ │ └── form.css │ ├── themes/ /* Temas */ │ │ ├── theme-a.css │ │ └── theme-b.css │ └── main.css /* Arquivo principal com imports */

✅ Boas Práticas de CSS

/* ✅ Agrupe propriedades relacionadas */ .button { /* Posicionamento */ display: inline-block; /* Dimensões */ padding: 12px 24px; /* Aparência */ background: #007acc; color: white; border-radius: 4px; /* Comportamento */ cursor: pointer; transition: background 0.3s ease; } /* ✅ Use modificadores para variações */ .button--large { padding: 16px 32px; font-size: 18px; } .button--danger { background: #dc3545; } /* ✅ Evite !important */ .button--disabled { opacity: 0.6; cursor: not-allowed; }

❌ Anti-patterns a Evitar

/* ❌ Não use IDs para estilos */ #my-button { background: red; } /* ❌ Evite seletores complexos */ .header .nav .menu-item .link:hover { color: blue; } /* ❌ Não crie dependências entre blocos */ .product-card .button { /* ❌ Acopla componentes */ margin-top: 10px; } /* ❌ Evite estilos genéricos demais */ .red-text { color: red; }

Ferramentas e Workflows

🛠️ Pré-processadores

Sass/SCSS: Permite nesting controlado e variáveis

// SCSS com BEM .product-card { background: white; &__title { font-size: 1.3em; } &__button { background: $primary-color; &--large { padding: 16px 32px; } } &--featured { border: 3px solid gold; } }

🔧 Linters e Validadores

Stylelint: Verifica conformidade com convenções BEM

CSS Comb: Ordena propriedades CSS de forma consistente

📦 Metodologias Complementares

SMACSS: Categoriza CSS (Base, Layout, Module, State, Theme)

ITCSS: Organiza CSS por camadas de especificidade

Performance e Manutenção

  • 💡 Otimize seletores: BEM naturalmente cria seletores eficientes
  • 💡 Remova CSS não utilizado: Use ferramentas como PurgeCSS
  • 💡 Versione seus componentes: Documente mudanças em componentes
  • 💡 Crie guias de estilo: Documente componentes e seus modificadores

Checklist de Qualidade

  • ✓ Todos os blocos têm nomes descritivos?
  • ✓ Elementos estão semanticamente corretos?
  • ✓ Modificadores são reutilizáveis?
  • ✓ CSS está bem organizado e comentado?
  • ✓ Não há conflitos de nomes de classes?
  • ✓ Componentes são independentes?
